Introduction

Welcome to Threat Detector®! This is an interactive and seamless virus hash and URL scanning service. A series of malware engines are used to scan the entry and determine the malicious status of the hash or URL. The brains of the service are linked with the VirusTotal API to scan the user’s entries. Features that the service provides are the ability to enter hashes and URLs for scanning, viewing the malicious status of a hash/URL, viewing the status of the vendors for each hash or URL, and viewing hash and URL meta data. API calls are made to scan the entered hash and URLs in order to provide a reliable analysis for the user’s benefit.

Design

The design uses a flask back-end that displays to a Bootstrap HTML front-end and stores hash and url data in an SQLite3 database. The back-end has all functioanlity to get the required properties (sha256, md5, sha1, etc...) from the database or API to display to the user. The VirusTotal API is called to get the results if the hash or URL has never been scanned before; otherwise, the results are pulled from the database. The back-end passes the results to the front-end to display to the user in an easy-to-read format.

Interpreting Displayed Information

After a user enters a hash or URL, a plethora of information is displayed. Upon clicking the "Scan" button, the user sees the number of vendors that flagged the entry as malicious, the user's entry, the malicious status, and the detection of individual vendor malware engines displayed under the "Detection" tab.

Detection Tab: This tab shows the number of vendors that scanned the user's entry. Each vendor has either have a check mark indicating that the vendor flagged the entry as clean (not malicious), an "X" mark indicating the vendor flagged it as malicious, and a "?" indicating the vendor cannot scan the hash/URL. A "?" from a vendor does not mean you should panic; it just means that some vendors do not understand the type of entry submitted and so does not produce verdicts for it.

Metadata Tab: There are two possible tab options depending on if the user enters a hash or URL. If a hash is entered, the user can see the size, MD5 hash, SHA256 hash, SHA1 Hash, and SSDEEP Hash under the "Hash Metadata" tab on the screen. If a URL is entered the user can see the domain's name, creation date, registrar, Forcepoint ThreatSeaker classification, Comodo Valkyrie Classification, Statvo Rank, and Majestic Rank under the "URL Metadata" tab on the screen.

Malicious Status Result and Tab: This tab displays whether the user's entry is malicious or not. There are three statuses the user can come across. The "NEGATIVE" status means that no malware was detected whatsoever. The "POSITIVE" status means that of the vendors that scanned the entry, 25% or more of the vendors flagged the entry as malicious. The "CAUTION" status means that less than 25% of the vendors flagged the entry as malicious; this status is more likely to represent false positives, so it is left up to the user to determine if you deem it as malicious or not.
